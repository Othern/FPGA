# FPGA 學習專案規範

## 學習者需求

- 起點：完全新手。
- 目標：在 6 週內完成可展示的 FPGA 專題。
- 投入時間：每週約 7 小時（建議 2 小時理論、3 小時實作與除錯、2 小時複習及紀錄）。
- 語言：以 SystemVerilog 為主，Verilog 為輔。
- 學習方式：理論搭配練習，最後完成整合專案。
- 語言與預算：中文為主；開發板與基本配件預算上限為新台幣 10,000 元。

## 優先學習範圍

1. 數位邏輯基礎
2. Verilog 與 SystemVerilog
3. 模擬與測試平台（testbench、assertion、波形除錯）
4. FPGA 開發流程、腳位約束與時序分析
5. UART、SPI、I2C 等通訊協定
6. 記憶體與 FIFO

## 建議環境與硬體

- 主要作業系統：Windows 10/11，便於使用 FPGA 廠商工具。
- 編輯器：VS Code。
- 前期模擬：OSS CAD Suite、Verilator 與 GTKWave。
- 開發板：優先考慮 Digilent Basys 3 或 Arty S7，購買前確認台灣當期價格在預算內。
- 前 3 週可只用模擬器；於第 3 至第 4 週之間取得開發板。

## 最終專題：UART 指令式資料緩衝器

電腦經 USB-UART 將資料或命令傳給 FPGA；FPGA 使用 UART 接收、將資料存入 FIFO，並依命令讀出資料或回傳狀態。

### 最小功能

- UART RX 每次接收一個 byte（位元組，即 8-bit 資料）。
- 將收到的 byte 寫入 16 或 32 深度的同步 FIFO。
- UART TX 從 FIFO 讀出並回傳資料，形成 byte echo（位元組回送）。
- 能正確處理 FIFO 的 empty/full 狀態。

### 標準功能

- 使用單一字元命令：`W` 寫入、`R` 讀出、`S` 查詢狀態。
- `S` 回傳 FIFO 使用量與 empty/full 狀態。

### 加分功能

- 支援 `WRITE 42`、`READ`、`STATUS` 等完整 ASCII 命令及錯誤處理。
- 以 LED 顯示 FIFO 狀態。
- 連接 SPI 感測器。
- 連接 I2C OLED 顯示器。

## 六週學習計畫與驗收

| 週次 | 主題 | 實作成果 | 驗收標準 |
| --- | --- | --- | --- |
| 1 | 二進位、布林代數、組合與循序邏輯、時脈與重設 | 組合邏輯、計數器、LED 閃爍器 | 能解釋組合／循序邏輯差異，並讀懂計數器波形。 |
| 2 | SystemVerilog：`module`、`logic`、`always_comb`、`always_ff`、參數化 | 可重用計數器、按鍵除彈跳、七段顯示器或 PWM | 寫出可綜合電路，設計不得以 `delay` 實現硬體功能。 |
| 3 | testbench、assertion、波形除錯、FSM | 自動化測試與簡單有限狀態機 | 測試避免時脈競爭，FSM 能通過正常與邊界情況測試。 |
| 4 | FPGA 流程、腳位與時脈約束、setup/hold、UART TX | LED／按鍵上板與 UART 傳送 | 能設定主時脈約束，並從 FPGA 傳送正確資料到電腦。 |
| 5 | UART RX、同步 FIFO、模組整合；CDC 風險辨認 | UART 收發與同步 FIFO | UART RX/TX、FIFO empty/full 及整合資料路徑皆通過模擬。 |
| 6 | 專題整合、硬體除錯、驗收與文件化 | UART byte echo + FIFO；單字元命令為進階 | 終端機可傳送資料並收到正確回傳，且 README 可重現操作流程。 |

## 六週後延伸方向

- 第 7 週：SPI 主機與感測器介面。
- 第 8 週：I2C 主機與 OLED 顯示。
- 完整 ASCII 命令解析器與非同步 FIFO 屬於進階內容，不列為六週結業必要條件。

## 開發原則

- 每個 RTL 模組須先有可重現的模擬測試，再進行硬體整合。
- 每週驗收須同時包含：能用自己的話解釋、能在執行前預測波形、能修改條件並同步更新 testbench。
- 使用 `always_comb` 撰寫組合邏輯，使用 `always_ff` 撰寫時序邏輯。
- 所有外部非同步輸入應考慮同步化；按鍵應處理除彈跳。
- 每個時脈都必須在約束檔中明確宣告；硬體驗證時應檢查時序報告。
- 專案中保留 README、模組介面說明、測試方式與驗收紀錄。
- 教材、批改與技術解釋中，首次出現專有名詞時應附上簡短的中文白話定義；必要時同時保留英文原文與縮寫，例如「時序分析（timing analysis）：檢查訊號能否在時脈要求的時間內穩定到達」。
