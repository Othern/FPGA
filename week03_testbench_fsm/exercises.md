# 第 3 週練習：先預測、再執行、最後修改

## 練習 1：手算波形

先寫出每拍狀態與 `detected`，再執行模擬：`0000`、`101`、`10101`、`1101001`。

## 練習 2：驗證測試能抓錯

暫時把 `GOT_10` 收到 `1` 的下一狀態改為 `IDLE`。先預測哪組測試會失敗，再執行確認，最後恢復 RTL。思考：只測一次 `101` 為何抓不到重疊處理錯誤？

## 練習 3：reset 邊界

送入 `10` 後拉低 `rst_n` 並跨過一個正緣，放開後再送 `1`。加入 assertion 檢查 reset 期間輸出為 0，且 reset 前後資料不得拼成命中。

## 練習 4：進階規格修改

將 DUT 改為偵測可重疊的 `1101`。先畫狀態圖，再同步修改 RTL 與 testbench。測試至少包含不命中、單次命中、共享前綴及中途 reset。

## 驗收清單

- [ ] 能用自己的話解釋三個狀態。
- [ ] 執行前完成至少兩組波形預測。
- [ ] 原始測試顯示 `PASS`。
- [ ] 刻意植入錯誤後，測試確實失敗。
- [ ] 恢復或修改 RTL 後，測試重新通過。
- [ ] 能在 GTKWave 指出第一個錯誤週期。
